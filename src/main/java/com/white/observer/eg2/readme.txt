观测者模式定义了对象之间的一对多依赖，当一个对象状态发生改变时，其依赖者便会收到通知并做相应的更新。
其原则是：为交互对象之间松耦合。
以松耦合方式在一系列对象之间沟通状态，我们可以独立复用主题（Subject）/可观测者（Observable）和观测者（Observer），
即只要遵守接口规则改变其中一方并不会影响到另一方。这也是其主要的设计原则。


下面是一个简单的气象站发送天气信息给布告板，然后布告板把天气信息显示在板上的例子。 

首先先建立三个接口，主题（Subject）、观测者（Observer）和显示内容（DisplayElement），分别代表气象站、布告板信息接收和布告板信息显示。
